正文（纯文本，可直接粘贴公众号）

Volcano 调度器的一轮调度，由 runOnce 创建 Session，在快照对象上执行 action 列表，再把可绑定任务交给异步绑定队列。 一、入口只负责注册 入口在 cmd/scheduler/main.go。两行空导入触发 pkg/scheduler/actions 和 pkg/scheduler/plugins 的 init，完成 action 与 plugin builder 注册。 注意，注册不等于执行。actions/factory.go 注册了 reclaim、allocate、backfill、preempt、gangpreempt、gangreclaim、enqueue、shuffle，但默认配置只启用 enqueue、allocate、backfill。 二、默认执行顺序由配置决定 DefaultSchedulerConf 在 pkg/scheduler/util.go。默认 action 链是 enqueue、allocate、backfill。 默认插件分两层。第一层是 priority、gang、conformance。第二层是 overcommit、drf、predicates、proportion、nodeorder。 这些插件在 OpenSession 时把函数注册进 Session。enqueue 主要用 QueueOrderFn、JobOrderFn、JobEnqueueable、JobEnqueued；allocate 和 backfill 主要用 JobValid、JobReady、Allocatable、PrePredicateFn、PredicateFn、NodeOrderFn。扩展点按 Tier 和声明顺序遍历，未注册对应函数的插件会被跳过。 三、每轮从 OpenSession 开始 调度周期默认是 1 秒，由 wait.Until(pc.runOnce, schedulePeriod, stopCh) 驱动。它不是墙钟意义上的整秒触发，而是 runOnce 返回后按 period 再等下一轮。 每轮 runOnce 会调用 OpenSession。OpenSession 先从 cache 做 Snapshot。Nodes、Queues、Jobs 在 Snapshot 中走 Clone；HyperNodes、分层集合与真实节点集合由 HyperNodesInfo 返回深拷贝。调度过程使用这一份对象，不直接使用 informer 后续写入的对象。 随后它按 Tier 顺序、Tier 内声明顺序执行 OnSessionOpen，完成插件函数注册。 四、enqueue 负责队列准入 enqueue 遍历 ssn.Jobs，把 Pending Job 按 Queue 分组。Queue 用 QueueOrderFn 排序，Job 用 JobOrderFn 排序。 当 PodGroup.Spec.MinResources 为空，或 ssn.JobEnqueueable(job) 通过时，enqueue 调用 ssn.JobEnqueued(job)，并把 PodGroup.Status.Phase 设置为 Inqueue。 五、allocate 是主要分配动作 allocate 负责给普通资源请求任务选择节点。默认配置包含 enqueue 时，仍是 Pending 的 Job 会被 allocate 跳过。 allocate 会按 QueueOrderFn、JobOrderFn、TaskOrderFn 组织优先队列，并在队列和作业之间循环推进。对单个任务，它先做 Allocatable，再做 PrePredicateFn 和 PredicateNodes。 BestEffort task 会在 allocate 中被跳过，后面交给 backfill 处理。 节点筛选后进入评分。Volcano 会先看当前空闲资源，soft sharding 下再考虑其他 shard；空闲资源不够时，才看 FutureIdle，也就是 Idle 加 Releasing 再减 Pipelined。 如果选中节点的 Idle 够用，任务进入 Allocate。如果只有 FutureIdle 够用，任务进入 Pipeline，表示本轮只记录对未来释放资源的预分配。 六、backfill 处理 BestEffort 任务 backfill 只处理 task.BestEffort 为 true 的任务。这个标记在 cache 构造 TaskInfo 时由 InitResreq.IsEmpty 设置。 backfill 会先跳过仍是 Pending 的 Job，然后在其余 Job 中挑 Pending 和 Pipelined 的 BestEffort task。Pipelined task 会先 UnPipeline，再进入 backfill 的待处理队列。 随后 backfill 执行 PrePredicateFn、PredicateNodes 和评分，然后调用 ssn.Allocate。这个路径不经过 backfill 自己的 Statement.Commit。ssn.Allocate 在 JobReady 后仍然会 dispatch，继续进入 AddBindTask 和异步绑定队列。 七、Pod Binding 是异步旁路 allocate 或 backfill 把可绑定任务写入 BindFlowChannel 后，Pod Binding 由 cache 的后台流程执行。 allocate action 使用 Statement 暂存 Allocate、Pipeline、Evict 等操作。只有 stmt.Commit 走到 Allocate 操作时，才会调用 s.allocate，再进入 cache.AddBindTask。 AddBindTask 会把任务状态更新为 Binding，并写入容量 5000 的 BindFlowChannel。cache.Run 里另有 processBindTask 每 20ms 读取这个 channel。 processBindTask 读到 batchNum 个任务会触发 BindTask。本轮读取结束后，只要本地 bindCache 非空，也会触发一次 BindTask。batchNum 默认是 1，可以通过 BATCH_BIND_NUM 调大。 BindTask 会新起 goroutine，先执行 PreBind 钩子，再调用 Binder.Bind。这一步通过 Kubernetes API 做 Pod Binding。成功后发 Scheduled 事件。失败后会标记 SchedulerError，执行 PreBindRollBack，并把任务放进 errTasks 等待 resync。 八、CloseSession 收尾 runOnce 最后 defer CloseSession。CloseSession 会先调用插件的 OnSessionClose，但这里遍历的是 ssn.plugins map，所以关闭顺序不保证等于打开顺序。 随后 JobUpdater 用 16 个 worker 并行更新 PodGroup 状态，updateQueueStatus 写回 Queue allocated，最后 cache.OnSessionClose 结束本轮会话。 下一轮 wait.Until 再次触发 runOnce，一切重新从新的 Snapshot 开始。
