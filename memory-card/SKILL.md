# 记忆卡片生成器

把一个源码机制(执行顺序、架构、调用链等)做成:**一张 HTML 绘制的记忆卡片 + 一篇可发布的短文**,装在同一个 HTML 页面里交付。

## 工作流程(必须按序执行)

### 第 1 步:源码分析与核实

**卡片上的每一个事实都必须有源码依据,禁止凭记忆编写。**

1. 用 Read/Grep 阅读目标源码,确认:名称拼写、执行/注册顺序、数值(权重、默认值、并发度)、feature gate 默认状态、版本信息
2. 顺序类内容要找到**决定顺序的那段代码**(如注册列表、循环体),不能只看单个定义
3. 在回复中先输出核实清单:每条事实附 `文件:行号`,有不确定的明确标注
4. 用户此前如果讨论过该主题,以本次重新核实的结果为准,纠正旧结论

### 第 2 步:风格选择

用 AskUserQuestion 让用户选择风格预设(每个预设同时决定**卡片视觉**与**标题/正文文案语气**),用户也可通过 Other 自定义。无法提问(自动化运行)或用户已表明偏好时,默认「工程蓝图」。

| 预设 | 卡片视觉 | 文案语气 |
|---|---|---|
| **工程蓝图**(技术导向) | 冷蓝灰主色、等宽字体、细网格/标注线质感、权重表格、精确旁注 | 严谨克制,术语用全称,关键处附源码 `文件:行号`,适合技术读者 |
| **生动叙事**(生动形象) | 暖色调(橙/珊瑚)、大圆角、拟物比喻图形(门禁、闸口、座位) | 把机制讲成一段旅程/故事,比喻贯穿(如"占座""落地"),标题抓人,面向泛技术读者 |
| **极简速查**(笔记卡) | 黑白 + 单一强调色、细边框、高信息密度、无装饰 | 要点清单式短句,几乎没有形容词,适合打印贴墙/复习 |
| **终端极客**(暗色) | 深色编辑器底、荧光强调色(青/绿)、代码块质感、行号细节 | 技术准确但口吻轻松,允许少量 geek 幽默,适合社区/博客 |

提问时给每个选项写一行 description 概括视觉 + 语气;选定后整张卡片和文案都要**贯彻该方向**,不要混搭成四不像。

### 第 3 步:生成 HTML 页面

以本 skill 目录下的 `template.html` 为骨架(先 Read 它),生成单文件 HTML:

- 把卡片内容写进 `<div id="card">`,使用模板预置的组件类(`.sec` 区块、`.kcard` 白卡、`.chip` 标签、`.flow-arrow` 流向箭头、`.side-card` 旁路卡、`.wtable` 权重表、`.diamond` 决策菱形、`.note` 旁注、`.return-path` 回路)
- 风格预设:在 `<body data-theme="blueprint|vivid|minimal|terminal">` 指定;模板内置四套 CSS 变量,但**变量只是底色**,装饰细节(字重层级、背景纹理、圆角、留白节奏)要按所选方向继续打磨
- 把发布物写进右侧面板:`#title-main` / `#title-alt1` / `#title-alt2`(主标题与两个备选,分开放、各自有复制按钮)、`#pub-body`(正文)
- 替换 `<title>` 和页脚来源说明(仓库、分支/版本、核实日期)
- **不要改动**模板的下载/复制 JS 和右侧面板结构

**文风规则(重要)**:

- **开篇直接进入主题**,第一句就是核心概念或结论,不写引子
- 禁止社交客套与 AI 套话:"分享给需要的朋友"、"希望对你有帮助"、"今天我们来聊聊"、"值得注意的是"、"让我们"一律不用
- 禁止中文破折号(——),改用逗号、句号或重组语序
- 陈述句为主;引导语只用克制的几种:"可以看到"、"注意"、"举一个例子"
- 英文术语、代码名两侧留半角空格(如 "由 PrioritySort 排序")
- **用户有自己的文章库或文风约定时,先读其中两三篇提炼风格再动笔**,以用户文风为准

**文案规范(公众号正文,重要)**:

- **纯文本**:公众号编辑器粘贴不支持 Markdown,正文禁止出现 `#`、`*`、`-`、`>`、反引号等标记,也不用【】▍■ 等装饰符号;小节用"一、二、三、"开头的独立短行分隔
- **中文全角标点**:,。:;?!()、——中文语境一律全角;英文单词、代码名、数字内部保持原样(如 v1.37、16 个 goroutine)
- **手机排版**:单段不超过 4 行(约 100 字),段间空行;关键结论可独立成段;列举优先写进句子里,不排竖列
- 目标是复制后**直接粘贴发布,零二次加工**
- 全角标点规则同样适用于**卡片上的中文文本**(标题、旁注、标签),代码名内部除外
- **不要依赖直接输出全角标点**(生成过程可能被规范化为半角):HTML 写完后必须运行本 skill 目录下的 `fix_punct.py` 做确定性转换,它会把紧邻中文的半角 `, : ; ? ! ( )` 转为全角,代码上下文不受影响:

  ```powershell
  python -X utf8 <skill目录>\fix_punct.py <HTML绝对路径>
  ```

**设计原则**(借鉴 frontend-design):

- 选定一个美学方向就全力精确执行,克制的极简和饱满的密度都可以,关键是"有意为之"
- 信息层级靠字号/字重/颜色拉开,不靠堆装饰;主色要有统治力,强调色要锐利
- 代码名一律等宽字体;中文标题可加 letter-spacing 提质感
- 禁止 AI 味套路:紫色渐变、无意义阴影堆叠、各风格趋同

**Logo 规则(重要)**:

- **禁止用 Unicode 图标字符当 logo**(⎈、☸、⚙ 等——渲染随系统字体漂移,几乎必错)
- 按优先级获取:
  1. **目标源码仓库内的官方 logo 文件**:先找 `logo/`、`docs/`、`assets/`、`art/` 等目录下的 SVG(如 kubernetes 仓库的 `logo/logo.svg`)
  2. **官方渠道下载**:仓库 README 引用的 logo URL、项目官网,用 curl 下载 SVG
  3. 都取不到:自绘简化 SVG 几何标识
  4. 最后兜底:缩写字 logo(主色底 + 白色等宽粗体,如 "K8s" "PG"),宁可朴素不可错误
- 官方 logo 用 **base64 data URI 的 `<img>` 内联**(`data:image/svg+xml;base64,...`),不要外链——外链图片会让 html2canvas 导出 PNG 失败或污染画布;logo 自带配色时把 `.card-logo` 背景设为 `transparent`

**连线/箭头规则(重要)**:

- 失败回路、旁路连线**不要手调像素**:给关键节点加 id,`load` 后用 `getBoundingClientRect` 相对 `#card` 计算坐标,动态注入(模板有 `layoutPaths` 骨架)
- 每条连线的两端都要有明确语义:起点贴出发元素的边,**终点必须接触目标元素的边界、箭头指向其内部**——指向区块间空白或边框外侧都算错
- **箭头指向哪个元素本身也是第 1 步要核实的事实**(失败后回到哪、重入经过哪个环节),不得为排版妥协;排版冲突用"错开进入高度"或"从两侧进入"解决,而不是换一个好画的目标

### 第 4 步:自检渲染

写完 HTML 后用无头 Edge 截图并 Read 检查:

```powershell
$edge = "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe"; if (-not (Test-Path $edge)) { $edge = "C:\Program Files\Microsoft\Edge\Application\msedge.exe" }
Start-Process -FilePath $edge -ArgumentList '--headless','--disable-gpu','--screenshot=<输出目录>\card-check.png','--window-size=950,2300','--default-background-color=FFFFFFFF','file:///<HTML绝对路径>' -PassThru -Wait
```

自检清单(逐项过):

1. 文字无溢出/截断,chip 无换行断字
2. **逐条核对每根箭头**:起点、终点、方向与语义一致(回队列的要真的"到"队列区块边上)
3. logo 形状正确(不是字体兜底的方块/雪花)
4. 各区块信息密度均匀,无大片空白或挤压
5. 风格统一:配色、圆角、字体贯彻所选预设
6. 中文标点为全角(已运行 `fix_punct.py`;可用其输出的 fullwidth-comma 计数佐证)

发现问题就改 HTML 重渲染,直到全部通过,再交付。

### 第 5 步:交付

- HTML 默认输出到用户桌面,命名 `<主题>-card.html`;用户指定了位置则从其指定
- 删除自检用的截图
- 回复中包含:核实结论摘要(含修正项)、所选风格、文件路径、一句使用说明(浏览器打开 → 左侧下载 PNG / 右侧复制文案)

## 注意事项

- 下载 PNG 依赖 CDN 加载 html2canvas,提醒用户需联网;PNG 为无损格式,模板默认 3 倍分辨率导出
- 主题不限于"执行顺序":架构分层、状态机、调用链、对比表均可,组件类自由组合
- 若用户只要卡片不要文章(或反之),右侧/卡片内容可精简,但页面结构保持不变
- 用户要求换风格时,只需改 `data-theme` 和文案重写,卡片结构可复用
